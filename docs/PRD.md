# 產品需求文件 (PRD) - 讀書筆記本系統

## 1. 專案概述
- **背景與動機**：在學校圖書館中，學生與教職員常常需要尋找特定的書籍，或是希望在閱讀後能夠記錄自己的心得與重點。為解決紙本紀錄不易保存且難以查找的問題，開發此「讀書筆記本系統」，旨在讓讀者能更快速地搜尋書籍，找到自己想要的書，並建立個人專屬的知識庫。
- **目標用戶**：學校圖書館的讀者（主要為學生與教職員）。
- **核心價值主張**：提供一個便捷且容易使用的系統，結合書籍檢索與個人筆記管理功能，讓求知者能快速找書、有效記錄，進而促進校園內的閱讀與知識回溯。

## 2. 功能需求

### 主要功能與使用者故事 (User Stories)
1. **可搜尋書籍功能**
   - 作為 **學校圖書館讀者**，我希望 **能夠透過關鍵字（如書名、作者）搜尋筆記本中的書籍**，以便 **更快速地找到我過去閱讀或想查詢的書**。
2. **記錄書名資訊**
   - 作為 **學校圖書館讀者**，我希望 **能夠手動新增書籍的基本資料（包含書名）進入系統**，以便 **完整追蹤我的閱讀實績**。
3. **撰寫閱讀心得**
   - 作為 **學校圖書館讀者**，我希望 **能夠針對特定書籍撰寫並修改長篇閱讀心得**，以便 **沉澱並記錄我對該書的全盤反思與感想**。
4. **重點摘錄**
   - 作為 **學校圖書館讀者**，我希望 **能夠以條列式記下書本中的特定佳句或核心知識點**，以便 **日後寫報告或複習時能馬上取得精華內容**。
5. **書籍評分機制**
   - 作為 **學校圖書館讀者**，我希望 **能為我讀過的書籍給予 1 到 5 星的評分**，以便 **我自己日後回顧，或提供他人決定是否借閱的參考**。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：Python 搭配 Flask 框架。
  - 前端顯示：HTML、CSS、JavaScript 搭配 Jinja2 模板引擎進行渲染。
  - 資料儲存：SQLite (適用於輕量並能快速佈署與開發的關聯式資料庫)。
- **效能與安全考量**：
  - **安全性**：防範一般的網頁漏洞，例如 SQL Injection（需使用 ORM 或參數化查詢）以及 XSS 攻擊（妥善將使用者的輸入利用 Jinja2 自動跳脫輸出）。
  - **效能**：資料庫設計時，應針對「搜尋」功能（如書名搜尋）建立適當的 Index，以提升查詢速度。即使在一般校園圖書館規模的讀取下，也能保證快速響應。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有 - MVP)**：
  - 新增書籍與記錄書名。
  - 基本的關鍵字搜尋功能。
  - 填寫並儲存閱讀心得的文字框。
  - 文字形式的重點摘錄區塊。
  - 數字化的 1~5 分評價欄位。

- **Should Have (應該有)**：
  - 依據評分高低或加入日期排序筆記列表。
  - 編輯與刪除書籍筆記的功能。
  - 簡單的標籤 (Tag) 功能，幫助書籍分類。

- **Nice to Have (可以有)**：
  - 簡單的使用者登入/註冊機制，將不同讀者資料隔離。
  - 結合網頁爬蟲或開源 API（如 Google Books）根據書名自動補齊書籍封面或作者資訊。
  - 能將心得與重點匯出成 PDF 或 Markdown 檔案。
